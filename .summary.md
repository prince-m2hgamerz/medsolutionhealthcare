## Goal
Replace all old custom Tailwind tokens (e.g., `text-ink`, `bg-canvas-light`, `border-hairline-light`, `text-shade-*`, `bg-aloe-*`, `bg-pistachio-*`) with new standardized tokens throughout admin panel pages and shared components.

## Constraints & Preferences
- Old token definitions in `tailwind.config.ts` remain untouched for backward compatibility; only usage in components/pages is replaced.
- Scope: all pages under `app/admin/` and shared components `components/admin/`.
- Already-migrated files must NOT be touched again.
- Reference file for new-token patterns: `components/admin/LeadsTable.tsx`.

## Key Decisions
- **Exact-hex direct replacements** (same color value, only token name changes):
  | Old Token | New Token | Hex |
  |---|---|---|
  | `text-shade-50` | `text-text-muted` | `#6B7280` |
  | `text-shade-40` | `text-text-light` | `#9CA3AF` |
  | `border-hairline-light` | `border-border` | `#E5E7EB` |
  | `bg-canvas` | `bg-surface` | `#F7F5F0` |

- **Semantic replacements** (similar purpose, slightly different hex):
  | Old Token | New Token | Old Hex → New Hex |
  |---|---|---|
  | `text-ink` | `text-text` | `#111827` → `#1C1C1E` |
  | `focus:border-ink` | `focus:border-border-focus` | `#111827` → `#1A6B8A` |

- **No exact new-token equivalent** — handle per case:
  - `bg-canvas-light` (`#F9FAFB`) → no new replacement of same hex. Best match: `bg-surface` (`#F7F5F0`).
  - `bg-canvas-cream` (`#FFFBEB`) → no new replacement. Best match: `bg-surface` (`#F7F5F0`).
  - `bg-shade-30` (`#D1D5DB`) → no new replacement. Best match: `bg-border` (`#E5E7EB`).
  - `text-shade-30` (`#D1D5DB`) → not in new system. Best match: `text-text-light` (`#9CA3AF`).

- **Green tokens (`aloe-*`, `pistachio-*`)** — keep as-is. The new system has no green palette. `accent-*` is gold, `primary-*` is blue. Green states (published, approved, super_admin) remain using old tokens by necessity.

- **Tokens still in new config** — no change needed:
  - `rounded-pill` is still `pill: "9999px"` in config
  - `text-caption`, `text-micro`, `text-body-md`, `text-heading-sm`, etc. are all still configured

- `accent-light` (`#FEF3CD`) is gold/amber, used for "new"/"converted" lead status. NOT interchangeable with `aloe-10` (green).
- `primary-tint` (`#E8F4F9`) is light blue, used for "contacted" lead status.
- `bg-surface` + `border-border` = card/detail panel pattern (see LeadsTable modal detail section).

## Critical Context
- Old tokens still compile and render; no breakage risk. Migration is purely for design-language consistency.
- No green tokens exist in the new system. `accent-*` = gold (spotlight), `primary-*` = blue (action/active). Green states (published blogs, approved testimonials, super_admin role) necessarily keep `aloe-10`/`pistachio-10`.
- The LeadsTable reference file uses: `bg-accent-light`/`text-accent-text` for gold badges, `bg-primary-tint`/`text-primary-mid` for blue badges, `bg-border`/`text-text-muted` for neutral badges, `border-border` for borders, `border-border-focus` for focus rings, `text-text` for body text, `text-text-muted` for muted/placeholder text, `text-caption` for label text.
